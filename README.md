# 写一个与动物学结合的agent项目

        ## Insights
        - 将项目定位为“动物学科普+检索增强Agent”可以在60小时内完成端到端闭环：小型数据集构建 → 检索 → Agent问答 → Web界面
- 与其做复杂多Agent编排，不如专注一个主Agent，辅以1个轻量“行为分析子Agent”（或在系统提示中用多角色模拟），更利于在有限时间内打磨体验
- 选择少量代表性物种（例如10–30个：常见城市鸟类、常见哺乳动物、典型濒危物种）即可体现“动物学结合”的深度，而不必追求大而全
- 可以通过在提示词中显式加入“若不确定则说明不确定并提示查阅文献”的规则，降低在动物学细节上的幻觉风险，同时展示科研态度
- 如果时间允许，可在知识库中加入“人类与该物种互动注意事项”（如不要投喂、保持距离等），让Agent具备“负责任的自然观察指导”特色

        ## Warnings
        - 动物学信息若完全依赖大模型记忆，容易出现物种分布、毒性、保护级别等方面的错误，必须用小型、可追溯的数据源做基础支撑
- 避免提供医疗或安全级别的建议（如被咬伤、被蜇伤的处理方式），应在系统中明确声明：本项目仅用于科普，不替代专业意见
- 数据抓取和使用图片时要注意版权与许可，建议优先使用开放数据源（如带有开放许可的物种列表），并在界面中标注来源
- 功能范围若扩展到复杂多Agent编排、跨平台App或大规模可视化，极易超出60小时与3人团队的实现能力，应控制在Web+单Agent主线
- 中文物种名与学名、英文名之间存在多对一和地区差异，若做名称映射需谨慎，避免在没有可靠映射表的情况下自动推断
